## 结论

需求 = 现有「AI 自动提取流水线」+ demo 的「人工标注工作台」合为一体。

- 需求第六块（AI 提取）：本项目已基本具备。
- 需求一～五（标注工作台、项目持久化、检索筛选、术语库管理）：本项目几乎全缺；demo 已实现其中绝大部分，可作产品基准。
- 真正的新开发集中在三件事：**数据模型升级、提取结果汇入标注库、统一模板导出与项目持久化**（见「集成专项」）。

## 集成专项（demo 也没有，需新开发）

1. **数据模型升级**：术语记录补 ukey、原文定位、审核状态、备注、修订时间。前提是管线保留「源行 → 术语」映射——当前 `run_pipeline` 只取文本列并去重（core/main.py:488-504），key 列在入口即丢失，需在加载、分块、提取结果回填全链路携带。
2. **提取结果 → 标注库汇入**：提取完成后术语以「未审核」状态进入标注工作台；外部术语表导入时与 AI 结果合并融合（冲突策略见「待确认 3」）。
3. **统一模板导出 + 项目持久化**：导出对齐 8 列中文模板；项目 JSON 含原文+术语+列绑定，一键复原。标注完成的术语表应可直接回流作下次提取的 glossary（闭环）。

## 实现注意

demo 的核心交互（划词加术语、A/Q/E 快捷键、单元格内高亮点击、双击编辑、吸顶筛选）在 Streamlit 中无法原生实现（无划词事件、无键盘钩子、表格只读）。建议标注工作台用独立 Web 前端（可直接改造 demo HTML）+ 本地后端提供提取 API，而非塞进现有 Streamlit 页面。
