把一个真实的角色名，
当成了乱编的词

古剑术语提取里发生的一件小事：程序自己判断错了、还自己动手删了，是同事一句提问发现的。记下来给大家提个醒。

日期 2026-06-15 项目 古剑 · 术语提取 影响 1 个词 状态 已修正

程序在抽术语时，把一个真实存在的角色名「武茗」，当成了"模型乱编出来的词"，并且自动把它删掉了。后来同事提了一个问题，发现判断有误——武茗是真的，只是原文里它中间多打了两个空格，程序没核对上。已修正，并改掉了"自动删"的做法。

那个词

问题出在两个看不见的空格

出错的词只有一个：武茗，一个游戏里的角色名。它在结果表里"来源"一栏是空的，程序就认为"原文里根本没有这个词，是模型编的"。但翻原文就会发现，它其实在第 560 行——

原文 · 第 560 行（角色名）
武茗

原文里这个名字，"武"和"茗"中间多打了两个空格（肉眼几乎看不出来）。

武茗 ← 模型把名字整理干净后的样子（正确）

问题在回头核对那一步：程序拿整理后的"武茗"去原文里找，但原文是"武  茗"（带空格），一对一比就对不上，于是误以为"原文里没有"——其实只是空格的差别。

同一批里别的角色名（三命、有头剑客、彩衣侯·空空子）都正常，只有武茗因为这两个空格栽了。"找不到来源"不等于"是模型编的"。

当时的对话

是同事一句提问，把它拦下的

下面是当时真实的来回（按发生顺序，略去技术细节）：

古剑术语提取 · 质检对话
Agent
判断（错）
武茗 是模型在一串角色名里编出来的词，原文里根本没有，该删。
Agent
自己动手
随后按这个判断，自动把武茗从结果里删掉了。
同事
一句提问
但这个词的出现，是因为查三遍都同时编出了同一个吗？这概率太小了吧。（程序对每条原文会独立查三遍，三遍都认出来才保留。三遍都凭空编出同一个名字，几乎不可能——所以它八成是真的。）
Agent
查证后认错
你说得对，是我错了——武茗 不是编的，是真词。原文 560 行里它中间多了两个空格。随后把删掉的词恢复，并改了做法。
顺带回答一个问题

那这件事，到底算不算"AI 幻觉"？

"AI 幻觉"通俗说就是：模型一本正经地说出原文或事实里其实没有的东西。照这个标准看——

武茗本身

不算幻觉

它真实存在于原文，模型只是把名字整理了一下。东西是真的，不是编的。

Agent 当时的判断

这才更像幻觉

"武茗是编的、该删"——这个结论说得很笃定，却没去翻原文核对，纯属没根据的猜测。

换句话说：程序冤枉了一个真词，说它是"乱编的"；而这个冤枉本身，才是真正没根据的。这也正说明，它的判断需要有人来核。

怎么处理的

已经修好，两步

1
让程序核对时不计较多余的空格。武茗就能正确对上第 560 行，"来源"一栏不再是空的。
2
把"找不到来源就自动删"改成"只做个标记、交人来看"。因为"找不到"不等于"是假的"——这种情况宁可标出来让人判断，也不让程序擅自删。
给我们的提醒

这件小事值得记一下

"查三遍都认"是个有用的信号

一个词能反复出现、扛过多轮检查，通常说明它是真的，不是噪音。看结果时可以多信这一点。

一个现象，可能有别的原因

"找不到来源"既可能是模型编的，也可能只是格式问题（这次是空格）。下结论前先翻一眼原始数据。

越是删除，越要人来拍板

判断错了还能改，删错了不一定救得回。涉及删除、覆盖这类操作，宁可让程序先标记、由人确认。

AI 能提速，但拍板和兜底还得靠人

同事那句"概率太小"，正是 Agent 容易跳过、而人一眼能察觉的常识。这次它直接拦下了一次本不该发生的删除。
