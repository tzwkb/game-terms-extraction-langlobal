"武茗"：当 Agent 把一个真词
当成了幻觉

一个自主执行的编程 Agent，把真实角色名误判为幻觉、还自动把它删了；最后是人凭一句"概率不对"的直觉拦下的。

Verdict · 一句话

Agent 把真词 误判成幻觉，还据此做了个会静默删真词的过滤器并已经删了；人的一句"三轮投票都幻觉？概率太小吧"，把方向从"删词"扳回了"查源 + 修匹配"。

定义校准 · What counts as a hallucination

那到底什么才算"幻觉"？

AI 幻觉＝模型自信、流畅地生成出不被输入或事实支撑的内容。两类：不忠实于来源（faithfulness — 输出在给定材料里查无实据）、违背事实（factuality — 编造现实里不存在的东西）。三个要件：是模型生成的 + 内容无据 + 姿态却自信。

武茗本身 · 不是幻觉

它有据、忠实于来源

武茗真实存在于源文件 560 行（武  茗）。模型只是把它正确归一并抽出——有据、忠实。"空源"是回溯匹配的 bug，不是模型凭空生成。

Agent 的诊断 · 这才是幻觉

它无据、却自信

"武茗不在原文、是模型在名字列表里脑补的武侠名"——流畅、笃定，却没查源文件就断言，完全不被数据支撑。正命中幻觉定义（严格说叫 confabulation，自信地虚构一个合理但错误的解释）。

真正的讽刺：Agent 指控一个真词是"幻觉"，而这次真正的幻觉，是这个指控本身。——模型能编造"一个真词不存在"，也能编造"它为何不存在的理由"，且都理直气壮。这正是它离不开人核的原因。

关键时刻 · Agent vs Human

谁错了，谁对了

Agent · 自主执行

两个错，都不轻

  1. 误判：把"空源"直接等同"幻觉"，没去查源文件就下结论——一段听起来很合理的叙述（"模型在一串名字里脑补了个武侠名"），整个是错的。
  2. 危险修复：基于错误前提，做了"空源即删"的过滤器，并已经把武茗从 248 删成 247。它会静默删掉每一个带格式空格的真词——是不可逆的数据破坏。
Human · 把关

一句直觉，决定性

"这个词出现，是因为三轮投票都出现了幻觉吗？这概率太小了吧。"

  1. 抓住三轮投票的统计含义：要 cnt≥3 才保留。若武茗是幻觉，等于三轮独立生成同一个生造名，概率极低。
  2. 于是"它能留下来"本身就成了"它是真的"的证据——直接推翻 Agent 的假设，把方向扳回查源。
修复 · The Fix

改对了方向：不是删词，是修匹配

① 让匹配容忍空格 —— core/main.py :: _batch_match_context。归一化掉空白再比子串，武茗 就能正确挂回 560 行。

core/main.py · _batch_match_context
-found = [term for term in remaining if term in text] # "武茗" in "武 茗" → False
+ntext = _ws.sub("", text) # 去掉空白
+found = [term for term in remaining if nterm[term] in ntext] # 归一后再比 → True

② 把"空源即删"改成"空源只标记" —— 因为空源可能是真词，绝不自动删，只在备注打 ⚠原文未命中,待核 交人工。

core/main.py · run_pipeline　/　cli.py
-drop_unsourced = True # 静默删除 → 会误删真词，危险
+flag_unsourced = True # 仅标记待核，never drop
~CLI: --no-flag-unsourced 可关
教训 · Takeaways

这件事告诉我们什么

本案里的 Agent 特指能自主读写代码、跑流程、做删改决策并直接落地的编程智能体，不是泛指的 AI/大模型。危险恰恰来自它的"自主执行"——一个只给建议、由人动手的 AI，不会造成同样的破坏。
1

三轮投票不止去噪

它还给出"存活＝可信"的信号。一个词能扛过三轮，通常是真的——读结果要会用这层统计含义。

2

症状 ≠ 单一成因

"空源"可以是幻觉，也可以是源文本的内部空格。下结论前先查原始数据，别被一段合理的叙述带走。

3

Agent 的"修复"可能更危险

一个自动删数据的过滤器，建立在错误前提上就是定时炸弹。越像模像样越要警惕。

4

不可逆动作 = 人来拍板

删除 / 覆盖 / 上线这类操作，必须先验证、再由人确认。宁可"标记待核"，不要"自动删除"。

★

Agent 编程提速，但拍板和兜底，仍离不开人

领域直觉（"概率太小"）正是 Agent 容易一步跳过、而人一眼能察觉的那层校验。人在回路不是形式——这次它直接拦下了一次数据破坏。
