01

这是什么

What it is

一个游戏术语的人工审校工作台。把要标注的原文 Excel 导入进来，在表格里挑出术语、填译文、做审核，最后导出一份标准 8 列术语表。它是术语提取工具的配套环节——AI 提取的候选术语，在这里完成人工把关。

原文.xlsx → 导入 · 标注 · 审核 → 术语表（8 列模板）.xlsx

① 导入原文

把待标注的 Excel 拖进来，原文显示在主表格。

② 标注审核

挑术语、填译文、设分类、标"已审核"。

③ 导出术语表

一键导出 8 列模板，交付或回流下一轮。

04

标准操作流程

Workflow
开始。首次使用直接点 📥 导入原文Excel；想清掉现有数据重来点 🔄 重置项目（确认后自动弹导入）；接着上次的活点 📂 加载项目。
导入原文。选择待标注的 Excel，表格随即显示原文；工具会按列名自动识别 原文 / 译文 / Key 列（识别不到才退回默认第 3 列）。
核对列。扫一眼 原文列 / 译文列 / Key列 是否选对；列名不规范、没自动识别时用下拉手动改——原文列选错，术语高亮会不准。
标注术语。四种加法见下一节；最常用：在原文里选中词按 A，或用 ☑️ 批量勾选添加 一次加多行。
审校。在右侧术语库改译文、设分类、点术语条上的 ✓ 标"已审核"；用状态/分类/排序/搜索管理。
（可选）合并已有术语表。📋 导入术语表 把旧术语并进来，可选 跳过 / 补空 / 覆盖。
导出 & 存档。📎 导出术语表 出 8 列模板；💾 保存项目 存成 .json，下次 加载项目 接着干。

四种加术语的方式

Adding terms
方式怎么操作适合
选中文本 + A在原文单元格里用鼠标选中一段词，按 A从句子里挑词
行首 ➕点表格每行最左的 ➕整行原文就是一个术语
➕ 新增术语右侧术语库点"新增术语"，手动输入表格里没出现、但要补录的词
☑️ 批量勾选添加按 Q 进入勾选模式 → 勾多行 → 回车 / 确定添加一次性把很多行加成术语
06

术语状态与颜色

Status & colors

原文里命中的术语会按状态高亮，一眼看出进度：

  • 黄 · 未审核　刚加进来、还没人把关的术语。
  • 蓝 · 已审核　已经人工确认过的术语。
  • 绿 · 译文命中　译文列里出现了该术语的译文（译文一致性参考）。

切换审核：在右侧术语条上点 ✓ 按钮，即可在 未审核 ↔ 已审核 之间切换；也可用状态下拉只看其中一类。

08

重置 / 加载 / 导入 别搞混

Reset vs Load vs Import

四个"放数据进来"的入口，区别在于对 原文 和 术语库 各做什么（鼠标悬停按钮也有同样提示）：

入口原文数据术语库（已标的术语）什么时候用
🔄 重置项目清空清空整个项目归零、彻底重开
📂 加载项目从 .json 恢复从 .json 恢复打开存档、接着上次干
📥 导入原文Excel载入 / 替换保留换一批原文继续标，术语库不丢
📋 导入术语表保留并入（跳过/补空/覆盖）把已有术语表合并进来
记一句就够：只有 🔄 重置项目 会清空你已经标好的术语库。想换原文但保留术语库，用 📥 导入原文Excel，别用重置。
09

保存与自动备份

Saving

💾 保存项目（手动）

导出一个 .json 存档，原文 + 术语库 + 分类全在内。这是正式存档，交接 / 换电脑就发这个文件。

📂 加载项目

读回上面的 .json，完整恢复现场。

⏱️ 自动备份（术语）

每隔几分钟把术语库暂存到浏览器，崩溃 / 误关后自动恢复。

注意：自动备份只存术语、不存原文。万一浏览器崩了，术语能恢复，但原文需要重新导入。正式保存请始终用 💾 保存项目。
10

常见问题

FAQ
数据会上传到服务器吗？
不会。全程在你本地浏览器内处理，不联网、不上传，适合处理机密原文。
关掉浏览器，标好的东西还在吗？
术语库有自动备份、重开能恢复；原文不会自动留存。要稳妥保存，务必用 💾 保存项目 导出 .json。
怎么交接给同事 / 换电脑继续？
把 保存项目 出的 .json 发给对方，对方用 📂 加载项目 打开即可，现场完整还原。
导出的术语表是什么格式？
标准 8 列 Excel 模板：Key值 · 术语分类 · 术语原文 · 术语译文 · 备注 · 来源原文 · 审核状态 · 最新修订时间。
术语高亮的颜色不对 / 没高亮？
多半是原文列没对上。导入时工具会按列名自动识别原文列，列名不规范时可能没认出来——到导入栏的"原文列"下拉，手动改成真正放原文的那一列即可。
