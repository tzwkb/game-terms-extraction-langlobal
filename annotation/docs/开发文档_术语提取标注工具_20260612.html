01

技术路径

Technical Approach

1.1 总体架构：父子双工具 + 文件契约

原文.xlsx → AI 提取工具（父项目） → 候选术语.xlsx · 8列模板 · 未审核 → 标注工具（子项目 · 人工审校） → 审定术语表.xlsx
审定术语表回流，作为父项目下一轮提取的参考术语库 —— 系统越用越准

父项目 · AI 提取工具

已有 Python + Streamlit 工程，本次只升级输入输出两端，提取管线不动。

子项目 · 标注工具

新开发。单 HTML 文件 + 原生 JavaScript，浏览器双击即用；唯一外部依赖为 Excel 解析库 SheetJS。

契约 · 8 列模板

两工具之间唯一接口，一张 xlsx 双向读写，互不依赖对方实现。

1.2 选型理由

  1. 标注的核心交互（划词、实时高亮、快捷键、行级跳转）只能在浏览器前端实现，任何后端语言都无法替代、只能附加——故子项目不引入后端；
  2. Streamlit 为脚本重跑模型，适合父项目"传参 → 跑批 → 看结果"，不适合字符级高频交互——故两工具分离；
  3. 单 HTML 交付：译员零安装（无 Python 环境 / 无打包 / 无服务器），数据全程不出本机（客户文本合规）。

1.3 性能技术路径（子项目）

设计容量目标：10 万行原文 × 3 万条术语全程流畅。demo v2.4 为全量渲染方案，大文件不可用，将重写渲染与检索层。
技术解决的问题
虚拟滚动（含滚动高度比例映射）只渲染可视区约 50 行，DOM 规模与文件大小无关；映射规避浏览器最大元素高度限制，行数可至 xlsx 格式上限 104.8 万
Aho-Corasick 多模式匹配术语高亮单遍扫描，成本与词表规模无关（替代巨型正则）
倒排索引（术语 ↔ 行号）跳转定位、导出回填由全表扫描改为查表
时间分片后台构建建索引 / 建词典切片执行（约 12ms/片让出主线程），界面不冻结，旧词典服务至新词典就绪
防抖 · 事件委托 · 单元格截断筛选打字不卡；监听器常数个；超长单元格防 DOM 膨胀
localStorage 崩溃草稿定时本地暂存术语库，崩溃可恢复

关键算法（AC 自动机、虚拟窗口计算）已做技术预研，可行性与性能验证通过；纯逻辑部分将配套 node 测试脚本（正确性 + 规模基准），不依赖浏览器即可回归。

1.4 父项目升级点

  1. 统一模板导出：管线结果增加转换函数，CLI 与 UI 同时落盘 results.xlsx（原始）与候选术语模板（8 列）；
  2. Key 列全链路携带：加载阶段保留 行Key↔文本 映射，穿过去重与分块，按术语上下文命中行回填 ukey；
  3. 列识别扩展：表头识别统一走 LLM（自包含），同次调用识别原文列、Key 列、双语英文列；识别到英文列自动预选双语模式；
  4. 回流兼容：8 列模板可直接作术语表输入（精确表头优先，避免"术语分类"误判为术语列）。

03

交互方式

Interaction Design

3.1 标注工具界面

左右双栏：左侧主表（原文全列表格，术语黄 / 蓝高亮、译文绿高亮，吸顶表头内嵌列筛选）；右侧术语库（计数、搜索、筛选、分页列表、逐条操作）。顶部两条工具栏：项目操作（新建 / 保存 / 加载 / 自动保存 / 行跳转）与数据操作（导入 / 列绑定 / 清空）。

3.2 核心交互流

导入原文 → 选 Key / 原文 / 译文列 → 表格渲染、高亮就绪
导入候选术语（提取工具产物）→ 选合并策略 → 术语入库并全表高亮
标注：划词按 A ／ 行首 ➕ 整行 ／ Q 批量勾选回车
审核：侧栏逐条 ✓ 切换；点击术语跳转原文核对，多处出现连点循环
导出 8 列模板 → 回流提取工具

3.3 快捷键

A 添加选中文本   D 删除   E 编辑   Q 批量模式   Enter 批量确认   Ctrl+S 保存   Esc 关闭弹窗  ｜  双击高亮词 = 编辑   双击行号 = 整行全文

3.4 提取工具侧

上传原文 + 术语表 → 自动列识别（可手动改）→ 开始提取（进度条 / 可取消 / 断点续跑）→ 完成后双按钮下载：标注模板（主） + 原始结果（辅）。

05

验收标准

Acceptance Criteria
  1. 需求清单（1–4 + 双语提取）逐条可演示；
  2. 性能：10 万行 × 3 万词——导入建索引 ≤10s（后台带进度）、滚动流畅、加词 / 筛选 / 跳转 <100ms（node 基准脚本可复跑）；
  3. 端到端闭环演示：提取 → 导入标注 → 审核 → 导出 → 回流提取。
06

风险与对策

Risks & Mitigations
风险对策
超大词表（>10 万）词典重建延迟时间分片后台构建，界面不冻结；旧词典先行服务
浏览器差异以 Chrome / Edge 为基准；滚动映射上限按最严浏览器取值
SheetJS CDN 需联网内网镜像，或后续改为内嵌（彻底离线）
崩溃草稿受 localStorage 配额限制（约 3 万词）超限自动降级跳过，保留手动 / 定时工程备份双保险
